# 第一次训练运行总结 — 农业植物叶片病害识别

> 运行时间: 2026年6月13日  
> 设备: NVIDIA GeForce RTX 4060 Laptop GPU (7.6 GB)  
> 框架: PyTorch 2.5.1 + CUDA 12.1

---

## 一、训练配置

| 配置项 | 取值 |
|--------|------|
| 模型 | ResNet-50 迁移学习 |
| 训练策略 | 冻结骨干网络，仅训练分类头 |
| 训练轮数 | 80 epochs |
| 初始学习率 | 0.001 |
| 学习率调度 | CosineAnnealingLR (退火至 1e-6) |
| 批次大小 | 32 |
| Dropout | 0.5 |
| 优化器 | AdamW (weight_decay=1e-4) |
| 损失函数 | CrossEntropyLoss + 类别权重 |
| 采样策略 | WeightedRandomSampler |
| 混合精度 | AMP (自动混合精度) |
| 数据增强 | 完整增强 (RandomCrop + Flip + Rotation + ColorJitter + Blur + Perspective + Affine) |
| 数据集划分 | 训练 70% / 验证 15% / 测试 15% (分层抽样) |
| 分类头结构 | Dropout → Linear(2048→512) → ReLU → BN → Dropout → Linear(512→256) → ReLU → Dropout → Linear(256→15) |

## 二、训练过程

训练稳定进行，未触发早停（最佳出现在 Epoch 66）。

### Loss 变化

|-------|-----------|----------|-----------|---------|
| 1 | 1.174 | 0.852 | 49.3% | 70.2% |

| 20 | 0.767 | 0.618 | 67.2% | 78.3% |

| **66** | — | — | — | **82.84%** ← 最佳 |
| 80 | 0.628 | 0.495 | 72.6% | 82.0% |

### 关键发现
- **学习速度快**: Epoch 1 验证准确率即达 70.2%，得益于 ImageNet 预训练权重
- **收敛较早**: 验证准确率在 Epoch 13 突破 79%，后续提升缓慢（Epoch 13→66 仅提升 3%）
- **无过拟合**: 训练准确率始终低于验证准确率（因数据增强 + Dropout 生效，训练时数据更难）
- **每 epoch ~33 秒**: 80 epochs 总耗时约 44 分钟

## 五、已知问题

### 1. Grad-CAM Hook 失败
- **原因**: 冻结骨干网络后，卷积层输出张量 `requires_grad=False`，导致 `register_full_backward_hook` 无法捕获梯度
- **状态**: 已修复（改用 `torch.autograd.grad` 直接计算）

### 2. 高置信度错误
- 63 个错误样本置信度 > 80%（模型"自信地犯错"）
- 主要集中在 Mosaic virus 类别的误判

### 3. 磁盘占用
- 21 个 checkpoint 文件占用 ~2.2 GB
- 建议: 后续只保留 best_model + 每 10 epochs 的 snapshot

### 4. 中文字体缺失
- Matplotlib 使用 DejaVu Sans 字体，中文标题显示为空白方块
- 影响: 仅图表显示效果，不影响数据

## 七、结论

首次运行采用 **ResNet-50 迁移学习 + 冻结骨干** 方案，在 PlantVillage 15 类别分类任务上达到 **83.07% 测试准确率**。

**亮点**:
- 训练稳定，无过拟合（数据增强 + Dropout 有效抑制背景过拟合）
- 大部分常见病害识别准确率 > 80%
- Yellow Leaf Curl Virus 识别精度达 99.8%

**不足**:
- 总体准确率 83% 低于预期的 90%+，核心瓶颈在于分类头训练不足
- 小样本类别（Mosaic virus 373 张、Potato healthy 152 张）表现差
- Grad-CAM 可视化待修复后生成

**下一步**: 修复 Grad-CAM → 运行二阶段微调 → 运行消融实验 → 撰写课程报告
