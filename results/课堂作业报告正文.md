## 摘要

农业植物叶片病害的早期自动识别是实现精准防治、减少农药滥用的关键技术。本文以 PlantVillage 公开数据集为研究对象，针对 15 类植物叶片病害分类任务，系统探索了基于迁移学习的深度学习方案。首先以 ResNet-50 冻结骨干 + CrossEntropy Loss 构建基线模型，测试准确率为 83.07%；进而引入二阶段微调策略和 Focal Loss，准确率跃升至 97.90%；随后对比了 Label Smoothing 与 Focal Loss、ResNet-50 与 ConvNeXt-Tiny 骨干网络的性能差异，单模型最佳准确率达到 98.26%；最后通过 MixUp 数据增强、Stage 级解冻优化和模型集成，将准确率进一步提升至 98.77%。通过系统消融实验，本文量化了二阶段微调（+14.83%）、损失函数选择（+0.36%）、骨干网络替换（−0.72%）、模型集成（+0.51%）等各技术组件的独立贡献。Grad-CAM 热力图分析验证了模型能够准确定位叶片病斑区域，具有较好的可解释性。实验结果表明，迁移学习结合二阶段微调是本任务最关键的性能驱动因素，而模型集成能够在单模型接近天花板时进一步获得增益。

**关键词**：植物病害识别；迁移学习；二阶段微调；ResNet-50；ConvNeXt；Grad-CAM；模型集成

## 第一章 引言

### 1.1 研究背景

农作物的病害是威胁全球粮食安全的重要因素之一。据联合国粮农组织估计，每年由植物病害造成的全球粮食产量损失高达 20%–40%。传统的人工目测诊断依赖植保专家的专业经验，存在效率低下、覆盖范围有限、主观性强等问题。随着计算机视觉和深度学习技术的快速发展，基于卷积神经网络（Convolutional Neural Network, CNN）的植物叶片病害自动识别已成为智慧农业领域的研究热点。

早期的植物病害识别研究主要依赖手工设计的图像特征（如颜色直方图、纹理特征、形状描述子等），结合传统机器学习分类器（如 SVM、随机森林）进行判别。这些方法在特定场景下取得了不错的效果，但手工特征的设计高度依赖领域知识，泛化能力有限。自 2012 年 AlexNet 在 ImageNet 大规模视觉识别挑战赛（ILSVRC）上取得突破以来，深度卷积神经网络已逐渐成为图像识别任务的主流方法。

PlantVillage 数据集是植物病害识别领域使用最广泛的公开基准数据集之一，包含 38 个类别、约 54,000 张实验室条件下拍摄的植物叶片图像。该数据集虽然采集条件较为理想化（均匀光照、纯色背景），但其类别间的高相似性和严重的类别不平衡问题，使其成为检验细粒度图像分类算法的良好平台。本文选取其中的 15 个类别（涵盖辣椒、马铃薯、番茄三种作物的常见病害及健康状态）作为研究对象。

### 1.2 研究目标与意义

本文的研究目标包括以下几个方面：

1. **构建高精度分类模型**：利用迁移学习技术，在 PlantVillage 数据集上训练能够准确识别 15 类植物叶片病害的深度学习模型，目标测试准确率超过 95%。

2. **系统对比训练策略**：通过消融实验，系统比较冻结骨干训练与二阶段微调、CrossEntropy Loss 与 Focal Loss 及 Label Smoothing、ResNet-50 与 ConvNeXt-Tiny 骨干网络等不同技术组合的性能差异。

3. **量化各组件贡献**：基于多次对照实验，量化二阶段微调、损失函数改造、骨干网络替换、数据增强（MixUp）、模型集成和测试时增强（TTA）各技术组件的独立贡献。

4. **验证模型可解释性**：利用 Grad-CAM 技术生成热力图，验证模型的分类决策是否真正基于叶片病斑区域，而非背景或其他无关特征。

本研究的学术意义在于提供了一套完整的迁移学习性能基准和消融分析框架，能够为后续同类研究提供参考；其应用意义在于验证了深度学习在植物病害自动诊断中的可行性，为智慧农业的实际部署提供技术支撑。

### 1.3 报告结构

本报告共分为七章。第二章介绍相关技术基础，包括卷积神经网络、迁移学习、关键训练技巧和模型可解释性方法。第三章详细阐述实验设计，涵盖数据集、模型架构、训练策略、实验矩阵和评估指标。第四章呈现各组实验的整体性能结果与训练过程分析。第五章以消融实验的形式，逐个量化各技术组件的贡献。第六章展示训练曲线、混淆矩阵、错误样本分析和 Grad-CAM 热力图等可视化结果。第七章总结全文工作，提炼关键发现，并讨论不足与后续改进方向。

## 第二章 相关技术基础

### 2.1 卷积神经网络

卷积神经网络（CNN）是深度学习中处理图像数据的核心架构。一个典型的 CNN 由三种基本层级结构堆叠而成：卷积层（Convolution Layer）负责提取局部特征，通过可学习的卷积核在输入特征图上滑动计算局部响应；池化层（Pooling Layer）通过下采样操作降低特征图的空间分辨率，增加感受野并控制计算量；全连接层（Fully Connected Layer）将提取的高层特征映射到最终的类别输出。

自 LeNet-5 于 1998 年首次成功应用于手写数字识别以来，CNN 架构经历了持续的演进。2012 年的 AlexNet 通过引入 ReLU 激活函数和 Dropout 正则化，首次在 ImageNet 上证明了深度 CNN 的巨大潜力。VGGNet（2014）采用连续的 3×3 小卷积核堆叠替代大卷积核，在简化设计的同时加深了网络深度。GoogleNet（2014）通过 Inception 模块的多分支并行卷积结构提升了多尺度特征的提取能力。

然而，简单地堆叠更多层会导致退化问题（Degradation Problem）：随着层数增加，训练误差不但不下降反而上升。He 等人于 2016 年提出的残差网络（Residual Network, ResNet）通过引入跳跃连接（Skip Connection），将层的目标从学习直接映射 $H(x)$ 转变为学习残差映射 $F(x) = H(x) - x$，即让网络层学习输入与输出之间的"差异"。这一简洁而深刻的创新使得训练超过 100 层的深度网络成为可能，ResNet 也因此成为计算机视觉领域最具影响力的骨干网络之一。

Liu 等人于 2022 年提出的 ConvNeXt 重新审视了 CNN 架构的设计空间。通过借鉴 Vision Transformer（ViT）的设计理念——包括使用大卷积核（7×7 Depthwise Convolution）、Layer Normalization 替代 Batch Normalization、GELU 激活函数替代 ReLU、以及倒置瓶颈结构——ConvNeXt 证明了纯 CNN 架构仍然能够在 ImageNet 上取得与 Swin Transformer 相媲美的性能，同时保留了 CNN 的简单性和高效性。

### 2.2 迁移学习

迁移学习（Transfer Learning）是解决小样本或计算资源受限场景下深度学习应用的核心技术。其基本思想是将模型在一个大规模源数据集（通常为 ImageNet，包含约 120 万张自然图像，1,000 个类别）上学习到的通用视觉表示迁移到目标分类任务上。

迁移学习背后的理论基础在于，CNN 的低层卷积核倾向于学习边缘、纹理、颜色等通用视觉基元，而高层卷积核则逐渐学习与任务相关的语义特征。ImageNet 预训练的低层特征对大多数自然图像理解任务都具有较好的迁移能力，因此在目标数据集规模有限时，"预训练 + 微调"（Pretrain + Fine-tune）范式能够显著优于从头训练的随机初始化方案。

常见的迁移学习策略包括：

- **冻结骨干（Frozen Backbone）**：固定预训练卷积层的所有权重，仅训练新初始化的分类头。该策略最保守，能够最大程度保留预训练知识，适用于目标数据集极小或与源域差异较大的场景。
- **全面微调（Full Fine-tuning）**：解冻所有层并使用较小的学习率对整体网络进行联合优化。该策略最激进，能够最大程度适应目标域，但风险在于可能破坏预训练特征（即"灾难性遗忘"）。
- **部分微调（Partial Fine-tuning）**：解冻网络的后若干层（通常为最后几个卷积阶段），保留低层通用特征不变，仅适配高层语义特征。该策略在知识保留和域适应之间取得折中。

本文采用"冻结骨干训练分类头 → 解冻骨干后若干层联合微调"的两阶段渐进策略，在保留预训练知识的同时精细化适配农业叶片病害的视觉特征。

### 2.3 关键训练技巧

**（1）损失函数**

损失函数定义了模型预测与真实标签之间的差异度量，是指导模型训练的核心信号。

- **交叉熵损失（CrossEntropy Loss）**：分类任务最常用的损失函数，其数学形式为 $\mathcal{L}_{CE} = -\sum_{c=1}^{C} y_c \log(p_c)$，其中 $y_c$ 为真实标签的 one-hot 编码，$p_c$ 为 Softmax 后的预测概率。交叉熵损失对所有样本施加相同的梯度尺度，在类别平衡的场景下表现良好。

- **Focal Loss**：由 Lin 等人在目标检测任务中提出，其核心思想是为困难样本（预测概率低）分配更高的损失权重，同时降低易分类样本（预测概率高）的权重。数学形式为 $\mathcal{L}_{FL} = -\sum_{c} y_c \cdot (1-p_c)^{\gamma} \cdot \log(p_c)$，其中 $\gamma$ 为聚焦参数。$\gamma=0$ 时退化为标准交叉熵；$\gamma$ 越大，对易分类样本的抑制越强。此外可结合 $\alpha$ 参数对类别权重进行平衡。

- **标签平滑（Label Smoothing）**：由 Szegedy 等人在 Inception-v3 中提出，将 one-hot 硬标签 $y = [0, ..., 1, ..., 0]$ 替换为软标签 $y_i^{LS} = (1 - \varepsilon) \cdot y_i + \varepsilon / C$，其中 $\varepsilon$ 为平滑系数。这一正则化技术通过降低模型对训练标签的绝对信任，防止模型过度自信，从而提升泛化能力和校准度。本文实验中取 $\varepsilon = 0.1$。

**（2）数据增强**

数据增强通过在训练时对原始样本施加随机变换来扩充有效训练集规模，提高模型的泛化能力和鲁棒性。本文采用的数据增强策略包括：

| 增强操作 | 参数 | 意义 |
|---------|------|------|
| RandomResizedCrop | 224×224 | 随机裁剪与缩放，模拟不同拍摄距离 |
| RandomHorizontalFlip | p=0.5 | 水平翻转，模拟不同视角 |
| RandomVerticalFlip | p=0.3 | 垂直翻转 |
| RandomRotation | ±30° | 随机旋转，模拟叶片角度变化 |
| ColorJitter | brightness=0.2, contrast=0.2, saturation=0.2 | 颜色抖动，模拟不同光照 |
| GaussianBlur | kernel=5, p=0.2 | 高斯模糊，模拟拍摄抖动 |
| RandomPerspective | p=0.3 | 透视变换 |
| RandomAffine | translate(0.1,0.1) | 仿射变换 |
| MixUp | α=0.2 | 两幅图像线性插值混合（第四次实验新增） |

MixUp（Zhang 等，2018）将两幅随机训练图像及其标签按 Beta 分布采样的系数进行线性插值，生成一个"虚拟"训练样本：$\tilde{x} = \lambda x_i + (1-\lambda) x_j$，$\tilde{y} = \lambda y_i + (1-\lambda) y_j$。这一方法通过构建平滑的样本间过渡区域来抑制模型对训练样本的"死记"，增强泛化能力。

**（3）学习率调度**

本文采用余弦退火调度（CosineAnnealingLR）：$\eta_t = \eta_{\min} + \frac{(\eta_{\max} - \eta_{\min})}{2} \cdot (1 + \cos(\frac{t}{T}\pi))$，使学习率从初始值 $\eta_{\max}$ 平滑退火至最小值 $\eta_{\min}$。该方法避免了阶梯式下降的突变，有助于模型在训练后期稳定收敛。

**（4）类别不平衡处理**

PlantVillage 数据集存在严重的类别不平衡问题：最大类别（Tomato Yellow Leaf Curl Virus）约 3,200 张训练图像，最小类别（Potato healthy）仅约 152 张。本文采用加权随机采样（WeightedRandomSampler），在构造每个训练批次时，以各类别样本数量的倒数作为采样权重，使每个类别在每个 epoch 中被模型"看到"的概率均等。

### 2.4 模型可解释性：Grad-CAM

深度学习模型常被诟病为"黑箱"，尤其在医疗、农业等关乎决策安全的领域，理解模型"为什么这样判断"至关重要。Gradient-weighted Class Activation Mapping（Grad-CAM, Selvaraju 等，2017）是 CNN 可解释性的经典方法。

Grad-CAM 的核心思想是：对给定的输入图像和目标类别 $c$，计算类别得分 $y^c$ 相对于最后一个卷积层输出特征图 $A^k$ 的梯度，并沿空间维度求平均，得到每个通道的重要性权重 $\alpha_k^c$：

$$\alpha_k^c = \frac{1}{Z} \sum_i \sum_j \frac{\partial y^c}{\partial A_{ij}^k}$$

随后以这些权重对特征图进行加权求和，并通过 ReLU 仅保留对目标类别有正向贡献的区域：

$$L^c_{Grad-CAM} = \text{ReLU}\left(\sum_k \alpha_k^c A^k\right)$$

最终将热力图上采样至原始图像尺寸并叠加显示，高亮区域即模型关注的"决策依据"。对于植物叶片病害识别任务，Grad-CAM 热力图应集中覆盖叶片的病斑和坏死区域，而非背景或健康叶面。

所有图像在输入模型前统一缩放至 224×224 像素，并归一化至 ImageNet 统计量（均值 [0.485, 0.456, 0.406]，标准差 [0.229, 0.224, 0.225]）。

### 3.2 模型架构

**骨干网络**：

- **ResNet-50**：包含 4 个卷积阶段的 50 层残差网络，约 25.6M 参数。移除原始分类头后，最后一层卷积输出 2,048 维特征向量。加载 PyTorch 官方提供的 ImageNet-1K V2 预训练权重。

- **ConvNeXt-Tiny**：包含 4 个阶段共 18 个 ConvNeXt Block 的现代化 CNN，约 28.6M 参数。最后一层输出 768 维特征向量。加载 PyTorch 官方提供的 ImageNet-1K V1 预训练权重。

**分类头**：

两个骨干网络共享相同结构的分类头：

```
Dropout(p=0.5)
  → Linear(embed_dim → 512)
  → ReLU
  → BatchNorm1d(512)
  → Dropout(p=0.5)
  → Linear(512 → 256)
  → ReLU
  → Dropout(p=0.5)
  → Linear(256 → 15)
```

该设计通过多层降维（embed_dim → 512 → 256 → 15）和多个 Dropout 层提供充足的容量和正则化，使分类头具备足够的学习能力，同时避免过拟合。

### 3.3 训练策略

本文统一采用**二阶段渐进训练策略**：

| 阶段 | 操作 | 学习率 | 早停耐心 | 说明 |
|------|------|--------|---------|------|
| 阶段一 | 冻结骨干，仅训练分类头 | 1×10⁻³ | 20 epochs | 让随机初始化的分类头快速收敛 |
| 阶段二 | 解冻骨干部分层，联合微调 | 1×10⁻⁴ | 10 epochs | 以更低学习率精细适配特征提取器 |

阶段一的目标是让新初始化的分类头在"冻结特征"的设定下学会组合预训练特征完成分类任务。阶段二则解冻骨干网络的后若干层，以 10 倍低于阶段一的学习率进行联合微调，在保留底层通用特征的同时，让高层语义特征向农业叶片病害域偏移。

对于 ResNet-50，解冻最后 30 个参数张量（对应 Stage 3 和 Stage 4 的大部分卷积层）。对于 ConvNeXt-Tiny，本文对比了参数级解冻和 Stage 级解冻两种策略，最终采用 Stage 级解冻（解冻最后一个 Downsample + Stage 4）。

|--------|------|

| 优化器 | AdamW (weight_decay=1×10⁻⁴) |
| 学习率调度 | CosineAnnealingLR (η_min=1×10⁻⁶) |
| 混合精度 | AMP (torch.cuda.amp) |

| 采样策略 | WeightedRandomSampler |

### 3.4 实验矩阵

### 3.5 评估指标

本文从多个维度评估模型性能：

- **总体准确率（Accuracy）**：整体分类正确的样本比例，即 $\text{Acc} = \frac{\text{正确分类样本数}}{\text{总样本数}}$。
- **宏平均指标**：先对每个类别计算 Precision、Recall、F1-Score，再对所有类别取平均。宏平均对各小类别给予与大多数类别同等的权重，适合衡量模型在全部类别上的综合表现。
- **各类别指标**：每个类别单独的 Precision（$\frac{TP}{TP+FP}$）、Recall（$\frac{TP}{TP+FN}$）、F1-Score（$\frac{2 \cdot P \cdot R}{P + R}$）。
- **混淆矩阵**：原始计数矩阵和行归一化矩阵，用于诊断模型在各类别之间的混淆模式。
- **高置信度错误**：预测概率 > 80% 但预测错误的样本，反映模型的过度自信程度。

## 第四章 实验结果与分析

### 4.1 整体性能对比

七组实验的测试集整体性能汇总如下表所示：

| 实验 | 测试准确率 | 宏平均 Precision | 宏平均 Recall | 宏平均 F1 | 错误样本数 |
|------|-----------|-----------------|-------------|-----------|-----------|
| E1 (基线) | 83.07% | 79.96% | 85.06% | 80.41% | 524 (16.9%) |
| E2 (Focal + 微调) | 97.90% | 97.66% | 98.40% | 98.01% | 65 (2.1%) |
| E3a (LS + ResNet-50) | **98.26%** | 98.05% | 98.67% | **98.35%** | ~54 (1.7%) |
| E3b (LS + ConvNeXt) | 97.54% | 96.74% | 98.12% | 97.38% | 76 (2.5%) |
| E4a (LS + MixUp) | 98.13% | 97.86% | 98.54% | 98.19% | 58 (1.9%) |
| E4b (LS + Stage 解冻) | 98.03% | 97.77% | 98.40% | 98.07% | 61 (2.0%) |
| **E4c (集成)** | **98.77%** | **98.80%** | **99.14%** | **98.96%** | ~38 (1.2%) |

从性能提升路径来看，模型经历了四个阶段：

```
基线 83.07% → 微调 97.90% → Label Smoothing 98.26% → 集成 98.77%
```

最大的单次提升出现在从 E1（冻结骨干）到 E2（二阶段微调）：**+14.83 个百分点**。后续的改进幅度逐渐减小——Label Smoothing 额外贡献 +0.36%，模型集成额外贡献 +0.51%——体现了逐渐逼近性能天花板的典型特征。

### 4.2 训练过程分析

**（1）阶段一收敛速度对比**

阶段一（冻结骨干训练分类头）中各实验的验证准确率演化展现出显著差异：

| 实验 | 损失函数 | Epoch 1 Val Acc | 阶段一最佳 Val Acc | 最佳出现 Epoch |
|------|---------|----------------|------------------|---------------|
| E1 | CE + 权重 | 70.2% | **82.84%** | 66 |
| E2 | Focal Loss | 63.5% | 78.77% | 26 |
| E3a | Label Smoothing | 66.2% | 78.32% | 24 |
| E3b | Label Smoothing (ConvNeXt) | **75.1%** | **90.15%** | 39 |

可以观察到：

- **E1（CE）的收敛速度最快**：Epoch 1 即达 70.2% 验证准确率，13 个 epoch 突破 79%，最终在 82.84% 达到最佳。这是因为标准交叉熵对不同难度的样本施以相同的梯度权重，模型可以快速"刷分"大量易分类样本。

- **E2（Focal Loss）和 E3a（Label Smoothing）的收敛速度慢于 CE**：两者阶段一最佳验证准确率（~78.3–78.8%）明显低于 CE（82.84%）。这不是因为模型"变差"，而是因为这些正则化损失函数降低了易分类样本的权重（Focal）或软化了标签（Label Smoothing），迫使模型在训练早期就关注困难样本。这种"抑制效应"虽然降低了阶段一的指标，却为阶段二的微调留出了更大的提升空间。

- **E3b（ConvNeXt-Tiny）的阶段性表现远超 ResNet-50**：阶段一最佳验证准确率达到 90.15%，比 ResNet-50 中最佳（CE, 82.84%）高出 7.31 个百分点。这说明 ConvNeXt-Tiny 的冻结特征本身就比 ResNet-50 更适配植物叶片分类任务，其现代化设计提取的特征具有更强的跨域泛化能力。

**（2）阶段二微调行为对比**

阶段二（解冻骨干微调）的验证准确率演化展现了各实验最终的收敛水平：

| 实验 | 阶段一 → 阶段二初始 | 阶段二 Epoch 1 | 阶段二最佳 | 微调提升 |
|------|-------------------|--------------|-----------|---------|
| E2 (ResNet-50 + Focal) | 78.77% | 91.08% | 98.32% | **+19.55%** |
| E3a (ResNet-50 + LS) | 78.32% | 88.79% | 98.32% | +20.00% |
| E3b (ConvNeXt + LS) | 90.15% | 92.73% | 97.58% | **+7.43%** |
| E4a (ResNet + LS + MixUp) | 76.80% | 86.49% | 98.48% | +21.68% |
| E4b (ConvNeXt + LS + Stage) | 90.79% | 92.63% | **98.06%** | +7.24% |

这些数据揭示了一个关键规律：**阶段一的特征质量与阶段二的收益呈反比关系**。

- ResNet-50 在阶段一仅表达出约 78% 的验证准确率，但解冻骨干后获得了 19–22 个百分点的巨幅提升。这表明 ResNet-50 的分类头受限于冻结特征的表达能力，但骨干网络本身可以很好地适配农业叶片图像——只是需要在目标域上"微调"高层特征才能释放其潜力。

- ConvNeXt-Tiny 在阶段一即达到 90% 以上的验证准确率（因为其特征本身已经足够好），但其微调收益仅为 7.2–7.4 个百分点。这并非缺点，而是因为"天花板效应"：从 90% 提升到 98% 所需的改进远小于从 78% 提升到 98%。

- **阶段二 Epoch 1 的跳变幅度**提供了两个骨干网络"学习难易度"的直观信号：ResNet-50 在解冻的瞬间获得了 10–12 个百分点的即时提升，而 ConvNeXt 仅为 1.8–2.6 个百分点。这进一步验证了 ResNet-50 特征对 PlantVillage 数据集的适配空间更大。

### 4.3 各类别性能分析

#### 基线模型（E1）的类别间差异

基线模型中各类别的 F1-Score 表现出极大的差异性：

- **表现优异（F1 > 85%）**：Yellow Leaf Curl Virus（94.4%）、Pepper healthy（96.4%）、Potato Early blight（93.0%）、Pepper Bacterial spot（87.2%）
- **表现中等（F1 70–85%）**：Potato Late blight（84.9%）、Tomato Bacterial spot（85.1%）、Tomato Late blight（82.1%）、Tomato Leaf Mold（83.4%）、Tomato Spider mites（81.1%）、Tomato healthy（83.3%）
- **表现较差（F1 60–75%）**：Tomato Early blight（70.9%）、Tomato Septoria leaf spot（74.7%）、Tomato Target Spot（72.3%）
- **表现极差（F1 < 60%）**：**Tomato Mosaic virus（50.5%）**、Potato healthy（66.7%）

Mosaic virus 的 Precision 仅为 33.7%（虽然 Recall 达 100%），意味着大量其他类别的样本被错误地判为 Mosaic virus，形成了一个"混淆黑洞"。根本原因在于该类别仅 56 个训练样本，在仅训练分类头的设定下，模型未从骨干特征中获得足够的区分信息。

引入二阶段微调后，各类别 F1-Score 实现了飞跃式提升和均衡化。以最佳单模型 E3a 和最佳集成 E4c 为例：

| Tomato Late blight | 82.1% | 96.3% | 98.4% | 🟢 |

E1 的核心问题是 **Mosaic virus 作为"混淆黑洞"吸引大量误报**——Top 10 混淆对中至少有 3 对与 Mosaic virus 有关，且 Precision 仅 33.7%。此外，不同病害（Septoria ↔ Early blight ↔ Late blight）之间的混淆也较为严重。

**E2（Focal + 微调，97.90%）的 Top 10 混淆对**：

## 第五章 消融实验与讨论

消融实验（Ablation Study）是本文的核心分析方法论。通过在保持其他因素不变的条件下逐个改变单一变量，我们可以隔离出每个技术组件对最终性能的独立贡献，并理解各组件之间的交互关系。

### 5.1 训练策略消融：冻结 vs 二阶段微调

| 实验 | 配置 | 测试准确率 | Δ |
|------|------|-----------|---|
| E1 | ResNet-50 + CE + 冻结 | 83.07% | — |
| E2 | ResNet-50 + Focal + 微调 | 97.90% | **+14.83%** |

**结论**：二阶段微调是本任务中最关键的单因素决策。解冻骨干网络的高层卷积层，使其从"通用自然图像特征"向"农业叶片病害特征"偏移，释放了预训练模型中被冻结的区分性信息。14.83 个百分点的提升远超任何其他组件的贡献（次大的组件贡献不超过 0.5%）。这一发现与计算机视觉领域的普遍共识一致：微调预训练模型几乎总是优于仅训练分类头。

### 5.2 损失函数消融：CE vs Focal Loss vs Label Smoothing

在相同的二阶段微调框架下，对比三种损失函数的最终效果：

| 损失函数 | 阶段一最佳 Val Acc | 最终测试准确率 | 分析 |
|---------|------------------|--------------|------|
| CE + 类别权重 | 82.84% | —（无微调） | 阶段一最高，但无微调对比 |
| Focal Loss (γ=2) | 78.77% | **97.90%** | 聚焦困难样本，微调后大幅提升 |
| Label Smoothing (ε=0.1) | 78.32% | **98.26%** | 平滑标签正则化，微调后最优 |

**有两个值得讨论的现象**：

**现象一：冻结阶段 CE > Focal Loss ≈ Label Smoothing**。CE 的初期收敛速度快于 Focal Loss 和 Label Smoothing，其在阶段一结束时的验证准确率高出 4–4.5 个百分点。原因在于：

- Focal Loss 通过 $(1-p_t)^\gamma$ 因子降低了易分类样本的损失贡献。在冻结骨干的场景下，绝大多数样本对预训练特征而言都是"易分类"的（Epoch 1 即 60–70% 验证准确率），Focal Loss 的大幅降权等同于减少了模型接收的有效梯度信号，导致收敛速度减缓。
- Label Smoothing 通过将硬标签 $y = [0,1,0]$ 软化为 $\hat{y} = [\varepsilon/C, 1-\varepsilon+\varepsilon/C, \varepsilon/C]$，人为引入了标签噪声。该噪声对模型有正则化作用，但也降低了优化目标的明确性。

**现象二：微调后 Label Smoothing 略优于 Focal Loss（+0.36%）**。这一差距虽然微小，但在实验条件一致（相同骨干、相同微调策略）的情况下具有一致性。可能的解释是：当微调让模型识别能力接近天花板后（98%+），Focal Loss 对困难样本的"聚焦"作用边际递减——因为"困难样本"已几乎不存在；而 Label Smoothing 对模型自信度的系统性抑制，反而带来了更好的泛化和校准效果。这与 Szegedy 等人提出的原始动机一致：Label Smoothing 通过防止模型对训练标签的过度拟合，提升在未见样本上的表现。

### 5.3 骨干网络消融：ResNet-50 vs ConvNeXt-Tiny

| 骨干 | 阶段一 Val Acc | 最终测试准确率 | 微调收益 | 可训练参数（微调阶段） |
|------|--------------|--------------|---------|---------------------|
| ResNet-50 | 78.32% | **98.26%** | +19.93% | ~10.5M (41.0%) |
| ConvNeXt-Tiny | **90.15%** | 97.54% → 98.03% | +7.43% → +7.24% | ~15.5M (54.6%) |

**ConvNeXt-Tiny 的"悖论"**：ConvNeXt-Tiny 的冻结特征远优于 ResNet-50（+11.83% 阶段一验证准确率），但最终测试准确率反而低了 0.23–0.72 个百分点。为什么"先天更好"的 ConvNeXt 在微调后反而不如 ResNet-50？

我们从三个角度理解这一现象：

1. **天花板效应**：ConvNeXt 的阶段一起点更高（90.15% vs 78.32%），意味着它离完美只有不到 10% 的提升空间，而 ResNet-50 有约 20%。对于 PlantVillage 数据集而言，98.3% 可能已经非常接近其本质上限（如标注噪声、不可分辨的边界样本），两个骨干网络最终都收敛到了相近的区间，差距更多来自它们在微调阶段的学习动力学，而非特征表达能力的根本差异。

2. **微调阶段的"学习空间"差异**：ResNet-50 使用 BatchNorm，在微调时 BatchNorm 统计量的适应（统计偏移）会引入额外的表示变化，这在某些任务上反而是有益的。ConvNeXt 使用 LayerNorm，其归一化行为不依赖批次统计，在小样本场景下更稳定，但在微调的"剧烈变化"阶段可能缺少 BatchNorm 带来的额外正则化/随机性。

3. **解冻策略的最优性**：ConvNeXt 的 4 个 Stage 具有不同的深度分布（Stage 3 有 9 个 Block 而 Stage 4 仅 3 个），不同级别的语义信息分布可能与 ResNet 不同。实验 B 通过引入 Stage 级解冻（仅解冻最后 2 个顶层模块，15.5M 参数），将 ConvNeXt 准确率从 97.54% 提升至 98.03%（+0.49%），验证了针对架构特性优化解冻策略的重要性。仍有进一步优化的空间——解冻更多的中间 Stage 或调整微调学习率的衰减策略，可能进一步缩小乃至消除与 ResNet-50 的差距。

**补充：TTA 对两种骨干的差异化影响**。测试时增强（TTA）对 ConvNeXt-Tiny 的提升（+1.07%，从 97.54% → 98.61%）显著大于对 ResNet-50 的预期提升（通常 < 0.5%）。这表明 ConvNeXt 的预测不确定性略高于 ResNet-50，但可通过多视图投票有效纠正。从实践角度，如果推理延迟不是瓶颈，ConvNeXt-Tiny + TTA（98.61%）是一个可替代 ResNet-50 单模型（98.26%）的可行方案。

### 5.4 数据增强消融：MixUp

| 配置 | 测试准确率 | 阶段二最佳 Val Acc | 高置信度错误 | 低置信度错误 |
|------|-----------|------------------|-------------|-------------|
| 无 MixUp (E3a) | **98.26%** | 98.32% | 若干 | — |
| MixUp α=0.2 (E4a) | 98.13% | **98.48%** | **0** | 30 |

MixUp 的引入呈现出一个有趣的**验证-测试背离**现象：验证集最佳准确率有所提升（98.32% → 98.48%），但测试准确率反而略微下降（98.26% → 98.13%）。这并非矛盾，而是反映了 MixUp 的正则化本质：

- **验证准确率更高**是因为 MixUp 的样本混合迫使模型学习更平滑的决策边界，这一平滑性在验证集（与训练集同分布）上表现为更稳定的预测。
- **测试准确率略低**可能是因为 MixUp 的平滑效应在 PlantVillage 这种"类间分明"的实验室数据集上并非关键需求。MixUp 的优势主要体现在存在域偏移或分布外样本的场景，而 PlantVillage 的测试集与训练集采集条件一致，域偏移极小。

**最有价值的发现是模型校准度的改善**：MixUp 模型的高置信度错误数量降为 **0**——模型在犯错时不再"自信地犯错"，而是给出较低的预测概率。这一特性在实际部署中非常重要：当模型"不确定"时（低置信度），系统可以触发人工复核流程，避免在缺乏把握的情况下做出错误判断。这种"自知之明"是模型可信赖性的重要维度，即便在准确率指标上未体现为提升。

### 5.5 解冻策略消融：参数级 vs Stage 级

ConvNeXt-Tiny 的顶层模块结构（共 8 个顶层子模块）与 ResNet-50 的连续参数张量结构存在根本差异：

```
ConvNeXt-Tiny 顶层模块（8 个）：
  features[0]: Stem (Conv2dNormActivation)
  features[1]: Stage 1 (3 ConvNeXt Blocks, 96 dim)
  features[2]: Downsample 1→2
  features[3]: Stage 2 (3 ConvNeXt Blocks, 192 dim)
  features[4]: Downsample 2→3
  features[5]: Stage 3 (9 ConvNeXt Blocks, 384 dim)
  features[6]: Downsample 3→4
  features[7]: Stage 4 (3 ConvNeXt Blocks, 768 dim)
```

| 解冻策略 | 具体模块 | 可训练参数 | 测试准确率 |
|---------|---------|-----------|-----------|
| 参数级 (num_layers=30) | 随机分布在 features[3]–[7] | ~43.8% | 97.54% |
| Stage 级 (unfreeze_stages=2) | features[6] + features[7] | 54.6% | **98.03%** |

Stage 级解冻的优势源于其**语义一致性**：每一组 Downsample + Stage 是一个完整的语义处理单元（感受野翻倍 → 在该感受野尺度下进行特征精炼）。按模块边界解冻保持了这些语义单元的内部一致性——微调时 Stage 4 的内部模块之间仍能"默契配合"。参数级解冻则可能在一个 Block 内部出现"部分神经元冻结、部分神经元可训练"的不一致状态，导致特征融合时出现信息断裂。

### 5.6 模型集成消融

| 集成方案 | 参与模型 | 测试准确率 | 宏平均 F1 | 模型间一致性 |
|---------|---------|-----------|-----------|-------------|
| 最佳单模型 | E3a (ResNet-50 98.26%) | 98.26% | 98.35% | — |
| 集成 V1 | E3a + E3b (ConvNeXt 97.54%) | **98.77%** | **98.96%** | **97.2%** |
| 集成 V2 | E3a + E4b (ConvNeXt 98.03%) | 98.55% | 98.72% | 97.4% |

这是一个违反直觉的结果：**集成 V1（含较弱的 97.54% ConvNeXt）** 的性能优于**集成 V2（含较强的 98.03% ConvNeXt）**，分别为 98.77% vs 98.55%。

解释这一"悖论"的关键变量是 **模型多样性（Diversity）** 而非个体准确率：

- **集成 V1**：两模型一致性 97.2%，意味着在 2.8% 的样本上意见不一致。在这些不一致样本上，概率平均使较"自信"的错误被另一个模型的正确预测"压制"，V1 成功修正了 ResNet-50 的 22 个错误和 ConvNeXt 的 48 个错误。
- **集成 V2**：两模型一致性更高（97.4%），意味着双方更倾向于在同一批样本上犯错（因为它们都在第三次实验中用相同的 Label Smoothing 训练，且 ConvNeXt 的微调更充分，与 ResNet-50 的"思考方式"更接近）。V2 仅修正了 ResNet-50 的 23 个错误和 ConvNeXt 的 29 个错误。

这一发现对模型的工程化部署具有实际指导意义：**当追求集成收益时，应优先选择架构差异性大的模型，而非一味挑选个体准确率最高的模型**。两个"中等但互补"的模型胜过两个"优秀但相似"的模型。

### 5.7 测试时增强（TTA）

| 模型 | 标准准确率 | TTA 准确率 | 提升 |
|------|-----------|-----------|------|
| ConvNeXt-Tiny (E3b) | 97.54% | **98.61%** | +1.07% |

TTA 对 ConvNeXt 的提升显著，使其 TTA 准确率（98.61%）超过了 ResNet-50 的单模型准确率（98.26%）。TTA 的基本原理是通过对同一张测试图像施加多种变换（如水平翻转、多裁剪），生成多个"视图"并使用模型的平均预测作为最终输出。这一技术通过"多角度看问题"减少单次推断的随机误差。ConvNeXt 受益更大的原因可能与其 LayerNorm 的小样本特性有关：在单张图像推断时，缺少批次统计的 LayerNorm 可能引入轻微的激活值抖动，多视图平均能够平滑这种抖动。

### 5.8 消融总结矩阵

| 消融变量 | 从 → 到 | 准确率变化 | 影响力评级 |
|---------|---------|-----------|-----------|
| **二阶段微调** | 冻结 → 微调 | 83.07% → 97.90% | ⭐⭐⭐⭐⭐ |
| **模型集成** | 单模型 → 双模型集成 | 98.26% → 98.77% | ⭐⭐ |
| **TTA** | 标准 → Simple TTA (2 views) | 97.54% → 98.61% | ⭐⭐ |
| **Stage 级解冻** | 参数级 → Stage 级 (ConvNeXt) | 97.54% → 98.03% | ⭐ |
| **Label Smoothing** | Focal Loss → Label Smoothing | 97.90% → 98.26% | ⭐ |
| **ConvNeXt 替换** | ResNet → ConvNeXt | 98.26% → 97.54% | ⭐ (负向) |
| **MixUp** | 无 → MixUp (α=0.2) | 98.26% → 98.13% | ~ (无效果) |

综合来看，**二阶段微调是决定性的性能驱动因素**，其贡献是其他所有技术组件之和的约 25 倍。当模型准确率超过 97% 后，各技术的边际贡献均在 0.5% 以内，这是逼近数据集性能上限的典型表现。此阶段更值得追求的目标从"提点"转向了"校准度改善"（如 MixUp → 零高置信度错误）和"鲁棒性增强"（如 TTA → 降低单次推断方差）。

## 第六章 可视化与可解释性

### 6.1 训练曲线

每次实验均生成了阶段一和阶段二的训练曲线，包含训练损失、验证损失、训练准确率和验证准确率四个子图。以下从训练曲线的角度汇总几个关键观察：

**（1）Loss 与 Accuracy 的对比**。所有实验均呈现"训练 Loss > 验证 Loss"和"训练 Acc < 验证 Acc"的特征模式。这是**数据增强生效的直接证据**：训练时模型面对的是经过了随机裁剪、翻转、颜色抖动、模糊等剧烈变换的样本，难度远高于"干净"的验证集样本；而验证时的评估仅经过标准化 Resize + CenterCrop，模型面对的是"最理想"的条件。这种"训练难、验证易"的模式是健康的——它表明数据增强有效地抑制了过拟合。

**（2）阶段一 → 阶段二的过渡**。训练曲线在阶段一切换到阶段二（解冻骨干）时呈现明显的**不连续跳跃**：训练 Loss 急剧上升（因为现在需要同时优化骨干参数），验证 Loss 急剧下降（因为骨干特征的表达能力被释放）。这一过渡是二阶段策略有效性的直观证据。

**（3）收敛速度与早停**。大多数实验的阶段二在 20–30 个 epoch 内达到最佳，随后进入平台期或轻微震荡。早停机制（patience=10）在 3 组实验中成功触发了提前结束训练，避免了无谓的算力消耗。

### 6.2 混淆矩阵

各组实验均生成了原始计数混淆矩阵和行归一化混淆矩阵。行归一化矩阵的对角线元素代表该类别的 Recall（召回率），非对角线元素代表混淆比例。

E1（基线）的混淆矩阵显示了"暗对角线"特征——Mosaic virus（第 14 类）和 Tomato healthy 的对角线亮度明显偏暗，而 Mosaic virus 列（误报集中地）呈现一条垂直亮线——大量其他类别的样本被误分为该类。

E2 及后续实验的混淆矩阵则呈现出"亮对角线"——所有类别的对角线元素均接近 1.0（行归一化后 > 95%），非对角线上的混淆信号稀疏且集中在语义相近的类别对之间（如不同番茄病害）。

E4c（集成）的混淆矩阵最为"纯净"：对角线元素几乎全部为 1.0（最优类别达 100% Recall），非对角线上的残余混淆基本消失，仅剩少量番茄内部病害间的交叉误判。

### 6.3 各类别准确率

各类别准确率柱状图直观对比了 E1 与最终模型（E3a/E4c）在 15 个类别上的表现差异：

- E1：仅 9 个类别 F1 超过 80%，3 个类别 F1 低于 75%（Mosaic virus 50.5% 为最低）
- E2–E4：全部 15 个类别 F1 超过 94%，8 个类别在集成方案中达到 100% F1

小样本类别（Potato healthy 23 测试样本、Mosaic virus 56 测试样本）的改善尤为显著，说明二阶段微调 + 加权采样策略的组合有效解决了类别不平衡对性能的负面影响。

### 6.4 错误样本分析

E1 的 524 个错误样本中，63 个为高置信度错误（> 80%），模型在大量样本上"自信地犯错"。错误样本的分布高度偏向 Mosaic virus 类别。

E2 的错误样本降至 65 个，其中高置信度错误仅 11 个。分析这些"幸存"的高置信度错误，发现其主要类型为：
- 同名病害跨作物混淆（Tomato Late blight ↔ Potato Late blight，视觉特征几乎相同）
- 番茄内部不同病害的极端相似病例

E4a（MixUp）的错误样本中高置信度错误降为 **零**，所有 58 个错误的置信度均在 80% 以下（其中 30 个 < 50%）。MixUp 虽然未提高准确率，但让模型学会了"诚实"：当遇到不确定的样本时，给出较低的预测概率而非假装自信。这种校准度的改善在实际应用中比 0.1% 的准确率差异更有价值。

### 6.5 Grad-CAM 热力图

Grad-CAM 可视化是本报告中唯一涉及模型可解释性的分析手段。技术上需要注意一个重要细节：

**实现挑战**：第一次运行时，Grad-CAM 生成失败。原因是冻结骨干网络后，卷积层输出张量的 `requires_grad=False`，导致 `register_full_backward_hook` 无法捕获梯度。解决方法是改用 `torch.autograd.grad` 直接计算目标类别得分对目标卷积层输出的梯度，绕过了 Hook 机制的限制。

生成的 9 张热力图覆盖了多种病害类型的代表性样本，观察结果如下：

- **✅ 正确的注意力模式**：对于辣椒细菌性斑点病（Pepper Bacterial spot），热力图高亮区域集中于叶片的褐色坏死斑点，而非周围的绿色健康组织或背景。对于番茄早疫病（Tomato Early blight），模型关注叶片上同心轮纹状的病斑区域。

- **✅ 跨病害的注意力特异性**：不同病害的热力图呈现不同的关注模式——疫病类（Late blight）关注大面积水渍状坏死区，斑点类（Septoria / Bacterial spot）关注散在的小型斑点，叶霉病（Leaf Mold）关注叶片正面的灰褐色霉层。

- **⚠️ 健康叶片的注意力弥散**：对于健康类别，Grad-CAM 热力图呈现较为弥散的分布（均匀覆盖叶片主体），这符合预期——模型通过"整体无病斑"来判断叶片是否健康，而非依赖某个特定的局部特征。

- **⚠️ 误判样本的注意力**：个别误判样本的热力图显示，模型将注意力放在叶片边缘或背景区域上（而非病斑区域），这可能解释了该样本被错误分类的原因。

Grad-CAM 分析从可解释性角度验证了两个事实：（1）模型的正确分类决策确实基于叶片病斑的病理学相关区域，而非背景或无关的视觉特征（如叶脉纹理、光照变化）；（2）模型的错误决策与注意力的错误聚焦相关，这为后续改进提供了明确方向（如引入注意力引导的损失函数）。

## 第七章 总结与展望

### 7.1 工作总结

本文系统探索了基于迁移学习的农业植物叶片病害识别方法。在 PlantVillage 15 类别分类任务上，通过 7 组对照实验和系统消融分析，实现了以下成果：

1. **基线验证**：以 ResNet-50 迁移学习 + 冻结骨干 + CrossEntropy Loss 构建基线，测试准确率为 83.07%，验证了迁移学习在该任务上的可行性。

2. **关键突破**：引入二阶段微调策略（先冻结训练分类头后解冻联合微调），将测试准确率从 83.07% 提升至 97.90%，提升幅度达 **14.83 个百分点**。这是本次研究中影响最大的单因素决策。

3. **精细化优化**：通过对比 Focal Loss 与 Label Smoothing，确认 Label Smoothing 在微调后略优（98.26% vs 97.90%）。通过对比 ResNet-50 与 ConvNeXt-Tiny，发现 ResNet-50 在微调后表现更优（98.26% vs 97.54%），并揭示了"阶段一起点更高但微调收益更小"的规律。

4. **消融完善**：MixUp 数据增强未能提升准确率但改善了模型校准度（零高置信度错误）；Stage 级解冻策略将 ConvNeXt 准确率从 97.54% 提升至 98.03%；模型集成达到最终最佳性能 **98.77%**。

5. **可解释性验证**：通过 Grad-CAM 热力图验证模型确实基于叶片病斑区域而非背景做出分类决策，增强了模型的可信赖性。

### 7.2 关键发现

1. **二阶段微调是最关键的操作决策**（+14.83%）。冻结骨干训练分类头可以让模型达到约 83%，但远未挖掘预训练模型的潜能。解冻骨干高层进行微调是本任务中投入产出比最高的改进。

2. **Label Smoothing 略优于 Focal Loss**（+0.36%），但两者的差距在微调后趋于缩小。损失函数的选择在微调阶段的边际效应远小于冻结阶段。

3. **ConvNeXt-Tiny 的反直觉表现**：其冻结特征远超 ResNet-50（阶段一 90.15% vs 78.32%），但微调后反而不如（98.03% vs 98.26%）。这一"悖论"的根源在于微调阶段的学习动力学差异和接近性能天花板后的收敛行为，而非特征表达能力的根本不足。

4. **模型多样性比个体准确率更重要**（集成实验）。一个 97.5% 的 ConvNeXt 比 98.0% 的 ConvNeXt 对 ResNet-50 的集成增益更大，因为其错误模式互补性更强。

5. **MixUp 的校准价值超过准确率价值**。在 PlantVillage 接近天花板的场景下，MixUp 未能提升准确率，但消除了高置信度错误——这一"隐形指标"在实际部署中具有更高价值。

### 7.3 不足与改进方向

1. **ViT 类骨干未探索**：本次实验使用了 ResNet 和 ConvNeXt 两种 CNN 类骨干，未涉及 Vision Transformer（ViT）或 Swin Transformer 等基于自注意力机制的架构。Transformer 在大规模预训练后展现出更强的视觉表示能力，可能突破 CNN 的性能天花板。

2. **SAM 优化器**：Sharpness-Aware Minimization（SAM）通过寻找损失平面的平坦最小值来提升泛化性能，在多个细粒度分类任务上取得了显著提升。本实验未采用，是一个可探索的方向。

3. **更高分辨率训练**：本文所有实验均在 224×224 分辨率下进行。植物叶片病害的区分有时依赖毫米级纹理细节（如霜霉病的孢子堆、锈病的孢囊），更高分辨率（如 384×384 或 512×512）可能对 ConvNeXt 和 Swin Transformer 等对大分辨率友好的架构有额外帮助。

4. **CutMix / RandAugment**：CutMix 通过对图像区域进行剪切-粘贴混合可能比 MixUp 更适合细粒度分类（因为它保留了局部纹理的完整性）。RandAugment 通过自动搜索最佳增强策略组合，可能比手工设计的增强管道更优。

5. **领域泛化验证**：PlantVillage 的实验室条件（纯色背景、均匀光照）与实际田间场景（复杂背景、多变光照、遮挡）存在显著差异。在当前模型上增加田间采集数据的迁移测试，是评估其在真实农业场景下实用性的必要步骤。

6. **报告局限性**：中文字体在 Matplotlib 中缺失（使用 DejaVu Sans），导致部分图表标题显示为空白方块，影响图表美观性但不影响数据分析本身。

### 7.4 课程收获

通过本次课程大作业的完整实践，深化了对以下核心概念的理解：

1. **迁移学习的完整流程**：从预训练权重加载 → 分类头设计 → 冻结训练 → 解冻微调 → 评估分析，涵盖了迁移学习在工程实践中的完整生命周期。

2. **消融实验的设计与分析**：通过控制单一变量逐步隔离各组件的贡献，学会了如何从"把所有技巧堆在一起效果好"的实践经验中，提炼出"每个技巧分别贡献了多少"的科学结论。

3. **超参数决策的上下文依赖性**：同一个技术（如 Focal Loss）在不同的训练策略下表现迥异（冻结阶段不如 CE，微调后与 LS 相当）。没有放之四海皆准的"最佳超参数"，只有"在此刻此条件下最合适的超参数"。

4. **Grad-CAM 的实现与调试**：通过解决 `register_full_backward_hook` 与 `requires_grad=False` 的兼容性问题，深入理解了 PyTorch 自动求导机制和 Hook 系统的工作方式。

5. **模型集成与多样性**：认识到模型集成中"多样性"的价值，理解了为什么两个"中等但互补"的模型可能胜过两个"优秀但相似"的模型。

# === 第一次训练：基线 ===

    --loss label_smoothing --finetune

    --tta --tta_mode simple

# MixUp 数据增强
python main.py --model transfer --backbone resnet50 \
    --loss label_smoothing --finetune --mixup --mixup_alpha 0.2

python main.py --model transfer --backbone convnext_tiny \
    --loss label_smoothing --finetune \
    --unfreeze_strategy stages --unfreeze_stages 2
