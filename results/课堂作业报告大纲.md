### 第三章：实验设计与方法（Methodology）

**3.1 数据集与预处理**

| 项目 | 说明 |
|------|------|
| 数据来源 | PlantVillage 公开数据集 |
| 类别数 | 15（3 种作物 × 各若干病害/健康） |
| 划分方式 | 训练 70% / 验证 15% / 测试 15%（分层抽样） |
| 图像尺寸 | 224×224 |
| 类别不平衡 | 最大类别（Yellow Leaf Curl）~3,200 vs 最小（Potato healthy）~152 |
| 数据增强 | RandomCrop + RandomHorizontalFlip + RandomVerticalFlip + RandomRotation + ColorJitter + GaussianBlur + RandomPerspective + RandomAffine |

**3.2 模型架构**

- **ResNet-50**：残差连接，50 层，约 25.6M 参数
- **ConvNeXt-Tiny**：现代化 CNN（Depthwise Conv + LayerNorm + GELU），约 28.6M 参数
- **分类头**：Dropout → Linear(2048→512) → ReLU → BN → Dropout → Linear(512→256) → ReLU → Dropout → Linear(256→15)

**3.3 训练策略**

| 阶段 | 操作 | 学习率 | 说明 |
|------|------|--------|------|
| 阶段一 | 冻结骨干，训练分类头 | 1e-3 | 让分类头适应下游任务 |
| 阶段二 | 解冻骨干部分层，联合微调 | 1e-4 | 用更低学习率微调特征提取器 |

**3.4 实验矩阵**

| 实验编号 | 骨干网络 | 损失函数 | 微调策略 | 特殊配置 |
|----------|---------|---------|---------|---------|
| Exp-1（第一次） | ResNet-50 | CE + 类别权重 | 仅冻结 | — |
| Exp-2（第二次） | ResNet-50 | Focal Loss | 二阶段微调 | — |
| Exp-3a（第三次） | ResNet-50 | Label Smoothing | 二阶段微调 | ε=0.1 |
| Exp-3b（第三次） | ConvNeXt-Tiny | Label Smoothing | 二阶段微调 | ε=0.1 |
| Exp-4a（第四次） | ResNet-50 | Label Smoothing | 二阶段微调 | MixUp α=0.2 |
| Exp-4b（第四次） | ConvNeXt-Tiny | Label Smoothing | Stage 级解冻 | unfreeze_stages=2 |
| Exp-4c（第四次） | 集成 | — | — | 概率平均 |

**3.5 评估指标**
- 总体准确率 (Accuracy)
- 宏平均 Precision / Recall / F1-Score
- 各类别 Precision / Recall / F1-Score
- 混淆矩阵
- 高置信度错误分析

**3.6 实验环境**
- GPU: NVIDIA GeForce RTX 4060 Laptop (7.6 GB)
- 框架: PyTorch 2.5.1 + CUDA 12.1
- 训练批次: 32，AMP 混合精度
- 优化器: AdamW (weight_decay=1e-4)

### 第四章：实验结果与分析（Results）

**4.1 整体性能对比**

| 实验 | 测试准确率 | 宏平均 F1 | 错误样本数 | 高置信度错误 |
|------|-----------|----------|-----------|-------------|
| Exp-1 | 83.07% | 80.41% | 524 (16.9%) | 63 |
| Exp-2 | 97.90% | 98.01% | 65 (2.1%) | 11 |
| Exp-3a | **98.26%** | **98.35%** | ~54 (1.7%) | 若干 |
| Exp-4c | **98.77%** | **98.96%** | ~38 (1.2%) | — |

**4.2 训练过程分析**

- 阶段一收敛速度：ConvNeXt (90.15%) > ResNet-50+CE (82.84%) > ResNet-50+Focal (78.77%) ≈ ResNet-50+LS (78.32%)
- 阶段二提升幅度：ResNet-50 (+19.55%) >> ConvNeXt (+7.43%)
- ConvNeXt 阶段一更强但微调收益更小
- 训练效率：每 epoch ~32–35 秒，完整训练 45–60 分钟

**4.3 各类别性能分析**

- 所有最终实验中 15 个类别 F1 均超 94%（vs 基线中仅 9 个 ≥ 80%）
- Mosaic virus：从 F1=50.5%（"混淆黑洞"）提升至 F1=100%（集成方案）
- Potato healthy：仅 23 测试样本，F1 从 66.7% → 100%
- 主要剩余混淆对：番茄不同病害间（Yellow Leaf Curl ↔ Bacterial spot），语义合理

**4.4 混淆矩阵对比**

- 第一次：Mosaic virus 吸引大量误报（Precision 33.7%）
- 第二次：混淆集中在同类作物病害间，语义合理
- 第四次集成：最大混淆对仅 10 次误判

### 第五章：消融实验与讨论（Ablation Study）

**5.1 训练策略消融：冻结 vs 二阶段微调**

| 配置 | 准确率 | Δ |
|------|--------|---|
| ResNet-50 + CE + 冻结 | 83.07% | — |
| ResNet-50 + Focal + 微调 | 97.90% | **+14.83%** |

> **结论**：二阶段微调是最关键的单因素提升，贡献超过所有其他技术之和。

**5.2 损失函数消融：CE vs Focal Loss vs Label Smoothing**

| 损失函数 | 阶段一最佳 Val Acc | 最终测试准确率 |
|---------|------------------|--------------|
| CE + 类别权重 | **82.84%** | 83.07%（无微调） |
| Focal Loss | 78.77% | 97.90% |
| Label Smoothing | 78.32% | **98.26%** |

> **结论**：冻结阶段 CE 更优，但微调后 Label Smoothing 略优于 Focal Loss（+0.36%）。Focal 和 LS 都在冻结阶段抑制"刷分"，但为微调留出更大提升空间。

**5.3 骨干网络消融：ResNet-50 vs ConvNeXt-Tiny**

| 骨干 | 阶段一 Val | 最终准确率 | 提升 | 微调收益 |
|------|-----------|-----------|------|---------|
| ResNet-50 | 78.32% | **98.26%** | +19.93% | 巨大 |
| ConvNeXt-Tiny | **90.15%** | 97.54% → 98.03% | +7.43% → +7.24% | 有限 |

> **结论**：ConvNeXt 冻结阶段更强（特征更好），但微调收益有限，最终不如 ResNet-50。搭配 TTA 可达 98.61%，弥补差距。Stage 级解冻比参数级解冻更优（+0.49%）。

**5.4 数据增强消融：MixUp**

| 配置 | 准确率 | 高置信度错误 |
|------|--------|-------------|
| 无 MixUp | 98.26% | 若干 |
| MixUp (α=0.2) | 98.13% | **0** |

> **结论**：MixUp 未提升准确率（-0.13%），但改善了模型校准度（零高置信度错误）。PlantVillage 已近性能天花板。

**5.5 模型集成**

| 方案 | 准确率 | 说明 |
|------|--------|------|
| 最佳单模型 | 98.26% | ResNet-50 + LS |
| 集成 V1 | **98.77%** | ResNet-50 (98.26%) + ConvNeXt (97.54%) |
| 集成 V2 | 98.55% | ResNet-50 (98.26%) + ConvNeXt (98.03%) |

> **结论**：模型多样性（diversity）比个体准确率更重要。集成 V1 两模型一致性更低（97.2% vs 97.4%），互补性更强，净收益更大。8 个类别达 100% F1。

**5.6 TTA（测试时增强）**

| 模型 | 标准准确率 | TTA 准确率 | 提升 |
|------|-----------|-----------|------|
| ConvNeXt-Tiny | 97.54% | 98.61% | +1.07% |

> TTA 对 ConvNeXt 的收益大于 ResNet，说明 ConvNeXt 预测不确定性略高但可通过多视图投票纠正。

**5.7 消融总结矩阵**

| 消融变量 | 从 → 到 | 准确率变化 | 影响力 |
|---------|---------|-----------|--------|
| 二阶段微调 | 冻结 → 微调 | 83.07 → 97.90% | ⭐⭐⭐⭐⭐ |
| Label Smoothing | Focal → LS | 97.90 → 98.26% | ⭐ |
| ConvNeXt 替代 | ResNet → ConvNeXt | 98.26 → 97.54% | ⭐ (负) |
| Stage 解冻优化 | 参数级 → Stage 级 | 97.54 → 98.03% | ⭐ |
| MixUp | 无 → 有 | 98.26 → 98.13% | ~ (无效果) |
| 模型集成 | 单模型 → 集成 | 98.26 → 98.77% | ⭐⭐ |
| TTA | 标准 → TTA | 97.54 → 98.61% | ⭐⭐ |

### 第七章：总结与展望（Conclusion）

**7.1 工作总结**
- 构建了 15 类植物叶片病害高精度分类模型
- 单模型最佳准确率 98.26%（ResNet-50 + Label Smoothing + 二阶段微调）
- 集成模型最佳准确率 98.77%（ResNet-50 + ConvNeXt-Tiny 概率平均）
- 通过系统消融实验量化了各组件贡献

**7.2 关键发现**
1. **二阶段微调是最关键因素**（+14.83%），远大于损失函数或骨干网络的选择
2. **Label Smoothing 略优**于 Focal Loss，但两者在微调后差距仅 0.36%
3. **ConvNeXt 冻结阶段更强**但微调收益有限，最终不如 ResNet-50
4. **模型多样性 > 个体准确率**：较弱的 ConvNeXt 与 ResNet-50 互补性更强
5. **MixUp 在近天花板场景无帮助**，但改善了模型校准度

**7.3 不足与改进方向**
- ViT / Swin Transformer 等 ViT 类骨干未探索
- SAM 优化器（Sharpness-Aware Minimization）可进一步提升泛化
- 更高分辨率（384×384）对 ConvNeXt 可能有额外帮助
- CutMix 可能比 MixUp 更适合细粒度分类
- RandAugment 可提供更系统的数据增强策略
- 中文字体缺失影响图表美观性（非技术问题）

**7.4 课程收获**
- 掌握了迁移学习的完整流程（冻结 → 微调 → 评估）
- 理解了损失函数选择与训练策略的交互作用
- 学会了通过消融实验系统分析各组件贡献
- 实践了 Grad-CAM 可解释性分析
- 积累了模型集成和 TTA 的实践经验

**B. 关键运行命令**
```bash
# 基线（第一次）
python main.py --model transfer --backbone resnet50 --epochs 80

# 二阶段微调 + Focal Loss（第二次）
python main.py --model transfer --backbone resnet50 --epochs 80 --finetune --loss focal

# Label Smoothing（第三次）
python main.py --model transfer --backbone resnet50 --loss label_smoothing --finetune
python main.py --model transfer --backbone convnext_tiny --loss label_smoothing --finetune

# MixUp + Stage 解冻 + 集成（第四次）
python main.py --model transfer --backbone resnet50 --loss label_smoothing --finetune --mixup
python main.py --model transfer --backbone convnext_tiny --loss label_smoothing \
    --finetune --unfreeze_strategy stages --unfreeze_stages 2

# TTA 评估
python main.py --eval_only --model transfer --backbone convnext_tiny --tta --tta_mode simple
```
