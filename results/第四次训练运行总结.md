## 三、实验 B：ConvNeXt-Tiny Stage 级解冻

> 动机：第三次总结明确指出 `num_layers_to_unfreeze=30`（参数张量级）对 ConvNeXt 不是最优值

### 代码改进

实现了全新的 `unfreeze_backbone(strategy='stages')` 方法，按骨干网络顶层模块解冻：

| 策略 | 说明 |
|------|------|
| `auto` | 根据骨干架构自动选择（ConvNeXt → stages, 其他 → last_n_params） |
| `stages` | 按顶层子模块解冻最后 N 个阶段（适用于 ConvNeXt/ResNet/EfficientNet） |
| `last_n_params` | 旧行为：按参数张量顺序解冻（对 ResNet-50 保持兼容） |

ConvNeXt-Tiny 顶层模块（共 8 个）：
```
features[0]: Stem (Conv2dNormActivation)
features[1]: Stage 1 (3 blocks, 96 dim)
features[2]: Downsample 1→2
features[3]: Stage 2 (3 blocks, 192 dim)
features[4]: Downsample 2→3
features[5]: Stage 3 (9 blocks, 384 dim)
features[6]: Downsample 3→4
features[7]: Stage 4 (3 blocks, 768 dim)
```

### 解冻对比

| 方案 | 解冻内容 | 可训练参数 | 占比 |
|------|---------|-----------|------|
| 旧 (30 参数张量) | 随机分布 | — | ~43.8% |
| 新 `--unfreeze_stages 1` | Stage 4 仅 | 14.3M | 50.4% |
| 新 `--unfreeze_stages 2` | Downsample + Stage 4 | **15.5M** | **54.6%** |
| 新 `--unfreeze_stages 3` | Stage 3 + Down + Stage 4 | 26.3M | 92.7% |

> 本次实验使用 `--unfreeze_stages 2`，仅解冻最后一个过渡层和 Stage 4，更精确地适配 ConvNeXt 架构。

## 八、结论

第四次运行完成了三项消融提升实验：

1. **MixUp 未受益**：在 PlantVillage 近天花板场景下，MixUp 的额外正则化无帮助（98.13% vs 98.26%）
2. **Stage 级解冻有效**：ConvNeXt-Tiny 从 97.54% → 98.03%（+0.49%），与 ResNet-50 差距缩至 0.23%
3. **模型集成最佳**：ResNet-50 + ConvNeXt-Tiny 概率平均达到 **98.77%**，F1 **98.96%**，8 个类别 100% F1

### 最终推荐方案

```
🏆 最佳单模型: ResNet-50 + Label Smoothing + 二阶段微调 → 98.26%
🏆 最佳集成:   ResNet-50 + ConvNeXt-Tiny 概率平均 → 98.77%
```

### 仍可探索的方向

| 方向 | 预期收益 | 说明 |
|------|----------|------|
| ViT/Swin Transformer 骨干 | +0.3~1.0% | 不同架构范式，可能突破 ResNet 天花板 |
| SAM 优化器 | +0.3~0.5% | Sharpness-Aware Minimization 提升泛化 |
| 更高分辨率 (384×384) | +0.2~0.5% | ConvNeXt 对高分辨率更敏感 |
| CutMix 增强 | 不确定 | 比 MixUp 更适合细粒度分类 |
| RandAugment | +0.1~0.3% | 更系统的数据增强策略 |
